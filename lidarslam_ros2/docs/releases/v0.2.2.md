# lidarslam_ros2 v0.2.2

This is the public `v2 beta` patch release for `lidarslam_ros2`.

Date: `2026-03-28`

## Highlights

- keeps the `0.2.1` public beta scope and refreshes it as a patch release
- fixes cross-distro CI drift so the public release path is green on both
  `Humble` and `Jazzy`
- keeps GNSS-weighted graph optimization, packet-path IMU deskew hardening,
  dynamic-object filtering, and classic-path fallback evidence aligned with the
  checked-in reports

## Public Snapshot

- fixed public entrypoint: `bash scripts/run_autoware_quickstart.sh`
- map-authoring summary: `python3 scripts/generate_map_authoring_report.py`
- place/classic closeout: `python3 scripts/generate_exploration_closeout_report.py`
- NTU VIRAL current default APE RMSE: `0.952 m`
- NTU VIRAL best observed APE RMSE: `0.870 m`
- MID360 current default cross-validation APE RMSE: `3.641 m`
- MID360 best observed cross-validation APE RMSE: `3.590 m`
- comparison page: `docs/comparison.md`

## Supported Scope

- ROS 2 LiDAR SLAM with a non-GPL default path
- pointcloud-map generation for Autoware-compatible workflows
- `map_projector_info.yaml` output as `Local` without GNSS and `LocalCartesian`
  with a stable GNSS origin
- release-readiness, map-authoring, dynamic-filter, and exploration-closeout
  reports from collected local artifacts

## Known Limits

- lanelet generation is out of scope
- MID360 evidence is still cross-validation against GLIM, not ground truth
- classic-path improvements remain fallback-path evidence, not the main public
  workflow
- place-recognition descriptor experiments are documented, but the public
  default still relies on the distance-based path

## Recommended Entry Points

```bash
bash scripts/run_autoware_quickstart.sh
bash scripts/run_rko_lio_graph_benchmark.sh
bash scripts/run_release_readiness_checks.sh --ape-threshold 0.10
python3 scripts/generate_map_authoring_report.py
```
